## 🎮 Как играть

### Для администратора (ведущего):
1. Откройте игру по ссылке
2. Нажмите "Начать игру" → "Создать комнату"
3. Передайте **код комнаты** участникам
4. Дождитесь подключения всех игроков
5. Нажмите "Начать игру"
6. Управляйте игрой через панель администратора

### Для игроков:
1. Откройте игру по ссылке
2. Нажмите "Начать игру"
3. Введите **код комнаты** от ведущего
4. Введите своё имя и бизнес
5. Нажмите "Присоединиться"
6. Отвечайте на вопросы каждого этапа

## ✅ Реальный режим (между разными устройствами)

Чтобы комнаты работали между разными компьютерами, нужен запущенный `socket.io` сервер.

1. Запустите backend:
```bash
cd server
npm install
npm run dev
```

2. На фронтенде укажите URL backend в `.env`:
```bash
VITE_SOCKET_URL=http://<IP_или_домен_сервера>:3001
```

3. Перезапустите фронтенд и откройте игру:
```bash
npm run dev -- --host 0.0.0.0 --port 8091
```

4. Проверьте backend:
```bash
http://<IP_или_домен_сервера>:3001/health
```
Должно вернуть `{"status":"ok"}`.

### Важно
- GitHub Pages хранит только фронтенд, поэтому без `VITE_SOCKET_URL` игра работает в демо-режиме (только внутри одного браузера).
- Для публичной игры разместите папку `server` на VPS/хостинге (например, ваш `game-system.tw1.ru`) и укажите этот домен в `VITE_SOCKET_URL`.

## 📦 Публикация обновлений

### Способ 1: Автоматический скрипт (рекомендуется)

```bash
./deploy.sh "Описание изменений"
```

Скрипт автоматически:
- Создаст коммит с вашими изменениями
- Отправит на GitHub
- Соберёт проект
- Опубликует на GitHub Pages

### Способ 2: Вручную

```bash
# Коммит изменений
git add .
git commit -m "Описание изменений"
git push origin main

# Публикация на GitHub Pages
npm run deploy
```

## 🎯 Структура этапов игры

1. **Ассортиминск** - Выбор: товар или услуга
2. **Брендск** - Баланс между брендом и ассортиментом
3. **Зачемск** - Зачем клиенты покупают?

5. **Цалово** - Целевая аудитория (B2B/B2C параметры)
